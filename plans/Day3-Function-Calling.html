Day 3：Function Calling - 工具调用与外部集成

欢迎来到AI技能学习的第三天！作为具备后端研发经验的工程师，今天我们将深入探索Function Calling（函数调用）这一核心技术，它使大模型能够与外部工具、API和数据源进行交互，极大地扩展了AI应用的能力边界。

一、概念引入：什么是Function Calling？

Function Calling（也称为Tool Calling）是一种让大语言模型智能调用外部函数或工具的能力。它不是让模型直接执行操作，而是让模型理解何时需要调用外部工具，并生成正确的调用参数。

Function Calling的核心价值在于：

  • 扩展模型能力：让大模型突破知识截止日期的限制，访问实时信息
  • 连接现实世界：集成API、数据库、计算工具等外部系统
  • 增强事实性：基于真实数据生成回答，减少幻觉
  • 标准化交互：提供结构化、可预测的工具调用接口

1.1 工作原理：三阶段执行模型

Function Calling遵循清晰的三阶段执行流程：

  1. 规划阶段：模型分析用户请求，决定是否需要调用工具，选择合适工具并生成参数
  2. 执行阶段：你的应用程序接收工具调用描述，实际执行外部函数并获取结果
  3. 响应阶段：模型将工具执行结果融入自然语言回答，呈现给用户

关键概念：模型只生成调用描述，不执行实际操作。这确保了安全性和可控性。

1.2 应用场景：五大典型用例

Function Calling在实际项目中广泛应用：

  • 工具调用：天气查询、计算器、单位转换等基础工具
  • API集成：连接CRM、ERP、电商平台等业务系统
  • 数据提取：从非结构化文本中提取结构化信息
  • 实时信息：获取股票价格、新闻动态、航班状态等
  • 复杂工作流：多步骤任务分解与协调

二、实现机制：函数描述与调用流程

2.1 函数描述规范（JSON Schema）

Function Calling使用JSON Schema描述函数接口，包含三个核心要素：

  1. 函数名称：唯一标识符，用于调用匹配
  2. 功能描述：自然语言说明，帮助模型理解何时使用此函数
  3. 参数规范：定义参数名称、类型、描述、是否必需

2.2 主流API格式对比

提供商 格式特点 调用响应
OpenAI 独立tool_calls字段，参数为JSON字符串 包含function_call对象的列表
Anthropic 内嵌在content中的结构化数据 tool_use类型的消息块
DashScope 兼容OpenAI格式，支持标准function calling 与OpenAI相同的结构

2.3 完整调用流程

一个标准的Function Calling流程包含以下步骤：

  1. 定义工具列表：创建函数描述JSON Schema
  2. 绑定工具到模型：通过API将工具描述发送给模型
  3. 模型决策：模型判断是否需要调用工具，生成调用参数
  4. 执行外部函数：应用程序解析参数，调用实际函数
  5. 返回结果：将执行结果格式化为模型可理解的格式
  6. 生成最终回答：模型基于工具结果生成自然语言响应

3.1 基础工具调用：天气查询

# Function Calling基础示例：天气查询工具
import json
from typing import Dict, Any
import requests

class QwenFunctionCallingDemo:
    """Qwen Function Calling演示类"""
    
    def __init__(self, api_key: str):
        """初始化Qwen客户端"""
        self.api_key = api_key
        self.base_url = "https://dashscope.aliyuncs.com/compatible-mode/v1"
        self.headers = {
            "Authorization": f"Bearer {api_key}",
            "Content-Type": "application/json"
        }
    
    def define_weather_tool(self) -> Dict[str, Any]:
        """定义天气查询工具"""
        return {
            "type": "function",
            "function": {
                "name": "get_current_weather",
                "description": "获取指定城市的当前天气信息",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "location": {
                            "type": "string",
                            "description": "城市名称，例如：北京、上海、广州"
                        },
                        "unit": {
                            "type": "string",
                            "enum": ["celsius", "fahrenheit"],
                            "description": "温度单位，摄氏度或华氏度"
                        }
                    },
                    "required": ["location"],
                    "additionalProperties": False
                }
            }
        }
    
    def call_qwen_with_tools(self, user_query: str, tools: list) -> Dict[str, Any]:
        """调用Qwen API（支持工具调用）"""
        endpoint = f"{self.base_url}/chat/completions"
        
        payload = {
            "model": "qwen-max",
            "messages": [{"role": "user", "content": user_query}],
            "tools": tools,
            "tool_choice": "auto",  # 自动判断是否调用工具
            "temperature": 0.3,     # 较低温度确保参数生成稳定
            "max_tokens": 500
        }
        
        try:
            response = requests.post(
                endpoint,
                headers=self.headers,
                json=payload,
                timeout=30
            )
            response.raise_for_status()
            return response.json()
        except requests.exceptions.RequestException as e:
            print(f"API调用失败: {e}")
            return None
    
    def execute_weather_function(self, location: str, unit: str = "celsius") -> str:
        """模拟执行天气查询函数（实际项目中会调用真实API）"""
        # 这里模拟返回固定数据，实际应调用天气API
        weather_data = {
            "北京": {"temperature": 15, "condition": "晴朗", "humidity": "45%"},
            "上海": {"temperature": 18, "condition": "多云", "humidity": "65%"},
            "广州": {"temperature": 25, "condition": "小雨", "humidity": "85%"}
        }
        
        if location in weather_data:
            data = weather_data[location]
            return json.dumps({
                "location": location,
                "temperature": data["temperature"],
                "unit": unit,
                "condition": data["condition"],
                "humidity": data["humidity"],
                "timestamp": datetime.now().isoformat()
            })
        else:
            return json.dumps({
                "location": location,
                "temperature": "未知",
                "unit": unit,
                "condition": "数据不可用",
                "error": "城市不支持"
            })
    
    def demo_basic_function_calling(self) -> None:
        """演示基础Function Calling流程"""
        print("=== 基础Function Calling演示 ===")
        
        # 1. 定义工具
        weather_tool = self.define_weather_tool()
        tools = [weather_tool]
        
        # 2. 用户查询
        user_query = "北京今天的天气怎么样？"
        print(f"用户查询: {user_query}")
        
        # 3. 调用Qwen API
        response = self.call_qwen_with_tools(user_query, tools)
        if not response:
            print("API调用失败")
            return
        
        # 4. 检查是否有工具调用
        message = response["choices"][0]["message"]
        if "tool_calls" in message:
            tool_call = message["tool_calls"][0]
            function_name = tool_call["function"]["name"]
            arguments = json.loads(tool_call["function"]["arguments"])
            
            print(f"模型决定调用工具: {function_name}")
            print(f"生成参数: {arguments}")
            
            # 5. 执行实际函数
            if function_name == "get_current_weather":
                location = arguments.get("location")
                unit = arguments.get("unit", "celsius")
                weather_result = self.execute_weather_function(location, unit)
                print(f"工具执行结果: {weather_result}")
                
                # 6. 将结果发送回模型获取最终回答
                second_payload = {
                    "model": "qwen-max",
                    "messages": [
                        {"role": "user", "content": user_query},
                        {"role": "assistant", "content": None, "tool_calls": [tool_call]},
                        {"role": "tool", "content": weather_result, "tool_call_id": tool_call["id"]}
                    ],
                    "temperature": 0.7
                }
                
                final_response = requests.post(
                    f"{self.base_url}/chat/completions",
                    headers=self.headers,
                    json=second_payload
                )
                
                if final_response.status_code == 200:
                    final_content = final_response.json()["choices"][0]["message"]["content"]
                    print(f"最终回答: {final_content}")
        else:
            print("模型决定不调用工具，直接回答")

2. 在Function Calling流程中，模型的角色是什么？

A. 直接执行外部函数并返回结果
B. 分析用户需求，生成工具调用参数
C. 管理应用程序的资源分配和调度
D. 验证工具执行结果的安全性
正确答案：B

解析：模型的角色是分析用户需求，决定是否需要调用工具，并生成正确的调用参数。

3. 以下哪种是Function Calling的正确JSON Schema结构？

A. {"name": "get_weather", "description": "天气查询", "params": ["city"]}
B. {"function": "weather", "args": {"city": "string"}, "required": true}
C. {"name": "get_weather", "description": "天气查询", "parameters": {"type": "object", "properties": {"city": {"type": "string"}}}}

正确答案：C

解析：正确的JSON Schema应包含name、description和parameters等字段。
