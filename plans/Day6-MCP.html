Day 6：MCP - 模型上下文协议

欢迎来到AI技能学习的第六天！作为具备后端研发经验的工程师，今天我们将深入探索MCP（Model Context Protocol）这一AI基础设施的关键协议，它实现了大模型与外部工具的解耦标准化，是构建可扩展AI应用的核心技术。

一、概念引入：什么是MCP？

MCP（模型上下文协议）是由Anthropic公司于2024年11月提出并开源的一种标准化通信协议，旨在统一大型语言模型（LLM）与外部工具、数据源之间的交互方式。

MCP的核心目标是解决AI集成中的"配置爆炸"问题：

  • 问题本质：在没有标准协议的情况下，每连接一个AI模型和一个外部工具都需要编写定制化集成代码，导致 m×n 配置爆炸
  • 解决方案：通过标准化协议，将集成复杂度从 O(m×n) 降至 O(m+n)
  • 价值主张：实现"一次开发，多次使用"，避免重复造轮子

1.1 MCP vs 传统集成：核心优势

与传统API/Function Calling相比，MCP具备五大核心优势：

  1. 解耦模型与工具：任意模型 ↔ 任意工具，无需硬编码绑定
  2. 标准化通用接口：统一的协议规范，降低学习成本
  3. 动态发现机制：AI模型无需预先知道可用工具，运行时协商
  4. 双向实时通信：基于JSON-RPC 2.0，支持长连接和持续会话
  5. 安全与权限控制：用户显式授权，凭证本地管理

1.2 应用场景：MCP的典型用例

MCP在实际项目中广泛应用：

  • IDE插件集成：在Cursor、Windsurf等IDE中动态调用外部工具
  • 桌面助手增强：Claude Desktop通过MCP连接本地文件系统、数据库
  • 多服务协作：AI Agent同时调用GitHub、Slack、本地日志等服务
  • 业务自动化：复杂工作流中多个工具的动态协调

二、核心架构：四大关键组件

2.1 MCP架构组成

MCP采用客户端-服务器架构，包含四个核心组件：

组件 角色 示例
MCP Host 运行LLM的应用程序环境，发起请求 Claude Desktop、Cursor IDE、Windsurf Editor
MCP Client 内置于Host中，负责与Server通信 每个Client与一个Server建立1:1连接
MCP Server 提供具体功能的服务端程序 GitHub MCP Server、Slack MCP Server、本地文件系统Server
Resources/Tools 实际被访问的数据或服务 Git仓库、Gmail邮箱、本地数据库

2.2 传输方式：两种标准机制

MCP支持两种标准传输机制，适应不同部署场景：

  1. stdio（标准输入/输出）：适用于本地工具（如文件系统、本地数据库），进程间通信
  2. SSE（Server-Sent Events）+ HTTP POST：适用于远程服务（如Slack、GitHub API），基于HTTP的实时通信

三、主流范式：与Function Calling的深度对比

3.1 MCP与传统Function Calling的区别

MCP代表了AI集成范式的根本转变：

对比维度 传统API/Function Calling MCP
耦合性 模型与工具强绑定，每对接需重写适配代码 解耦设计，任意模型 ↔ 任意工具，标准化接口
集成成本 O(m×n)复杂度，每增加工具需适配所有模型 O(m+n)复杂度，工具一次实现，全局复用
发现机制 工具列表需硬编码，无法动态扩展 动态发现，运行时协商，支持热插拔
通信模式 请求-响应（无状态），适合简单交互 持续会话（有状态上下文），支持复杂多轮交互
标准化程度 厂商私有协议，生态碎片化 开源开放标准（Apache 2.0许可），生态统一

3.2 安全机制：多层防护设计

MCP通过多层安全机制确保AI应用的安全性：

  1. 用户同意与控制：用户必须明确同意所有数据访问和操作
  2. 数据隐私：主机必须在向服务器公开用户数据前获得用户同意
  3. 工具安全：工具代表任意代码执行，需谨慎对待，用户授权前需了解功能
  4. LLM采样控制：用户必须明确批准任何LLM采样请求

4.2 MCP工具定义与调用

# MCP工具定义与调用示例
import json
from datetime import datetime

class MCPTools:
    """MCP工具定义示例"""
    
    def __init__(self):
        self.tools = [
            {
                "name": "weather",
                "description": "获取指定城市的天气信息",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "location": {"type": "string", "description": "城市名称"},
                        "unit": {"type": "string", "enum": ["celsius", "fahrenheit"], "description": "温度单位"}
                    },
                    "required": ["location"]
                }
            },
            {
                "name": "calculator",
                "description": "数学计算工具",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "expression": {"type": "string", "description": "数学表达式"}
                    },
                    "required": ["expression"]
                }
            }
        ]
    
    def execute_tool(self, tool_name: str, arguments: dict) -> str:
        """执行工具调用"""
        if tool_name == "weather":
            return self.get_weather(arguments.get("location"), arguments.get("unit", "celsius"))
        elif tool_name == "calculator":
            return self.calculate(arguments.get("expression"))
        else:
            return json.dumps({"error": f"工具不存在: {tool_name}"})
    
    def get_weather(self, location: str, unit: str = "celsius") -> str:
        """模拟天气查询工具"""
        weather_data = {
            "北京": {"temperature": 15, "condition": "晴朗", "humidity": "45%"},
            "上海": {"temperature": 18, "condition": "多云", "humidity": "65%"},
            "广州": {"temperature": 25, "condition": "小雨", "humidity": "85%"}
        }
        
        if location in weather_data:
            data = weather_data[location]
            return json.dumps({
                "location": location,
                "temperature": data["temperature"],
                "unit": unit,
                "condition": data["condition"],
                "humidity": data["humidity"],
                "timestamp": datetime.now().isoformat()
            })
        
        return json.dumps({
            "error": "城市不支持",
            "location": location,
            "suggestion": "请检查城市名称"
        })
    
    def calculate(self, expression: str) -> str:
        """计算器工具"""
        try:
            # 安全评估数学表达式
            allowed_chars = "0123456789+-*/(). "
            if any(char not in allowed_chars for char in expression):
                return json.dumps({"error": "表达式包含不安全字符"})
            
            result = eval(expression)
            return json.dumps({
                "expression": expression,
                "result": result,
                "timestamp": datetime.now().isoformat()
            })
        except Exception as e:
            return json.dumps({
                "error": f"计算失败: {e}",
                "expression": expression
            })

# MCP客户端调用示例
class MCPClient:
    """MCP客户端实现示例"""
    
    def __init__(self, mcp_server_url: str):
        self.server_url = mcp_server_url
        self.tools = MCPTools()
    
    def send_request(self, request: dict) -> dict:
        """发送MCP请求"""
        # 简化实现，实际应使用JSON-RPC 2.0协议
        if "method" not in request:
            return {"error": "缺少method字段"}
        
        method = request["method"]
        params = request.get("params", {})
        
        if method == "tools/list":
            return {
                "result": self.tools.tools,
                "jsonrpc": "2.0",
                "id": request.get("id", 1)
            }
        elif method == "tools/call":
            tool_name = params.get("name")
            arguments = params.get("arguments", {})
            
            result = self.tools.execute_tool(tool_name, arguments)
            return {
                "result": json.loads(result),
                "jsonrpc": "2.0",
                "id": request.get("id", 1)
            }
        else:
            return {
                "error": {"code": -32601, "message": "方法不存在"},
                "jsonrpc": "2.0",
                "id": request.get("id", 1)
            }

# 使用示例
if __name__ == "__main__":
    client = MCPClient("http://localhost:8080")
    
    # 获取可用工具列表
    tools_request = {
        "jsonrpc": "2.0",
        "id": 1,
        "method": "tools/list"
    }
    
    tools_response = client.send_request(tools_request)
    print("可用工具:", json.dumps(tools_response, indent=2, ensure_ascii=False))
    
    # 调用天气工具
    weather_request = {
        "jsonrpc": "2.0",
        "id": 2,
        "method": "tools/call",
        "params": {
            "name": "weather",
            "arguments": {"location": "北京"}
        }
    }
    
    weather_response = client.send_request(weather_request)
    print("天气查询结果:", json.dumps(weather_response, indent=2, ensure_ascii=False))

1. MCP（模型上下文协议）的核心设计目标是什么？

A. 提高大模型的推理速度和准确性
B. 统一大模型与外部工具的交互，降低集成复杂度
C. 替代所有现有的API接口标准
D. 强制所有AI应用使用相同的编程语言
正确答案：B

解析：MCP的核心目标是统一大模型与外部工具的交互方式，通过标准化协议将集成复杂度从O(m×n)降至O(m+n)，解决"配置爆炸"问题。

2. MCP架构中的"Host"组件主要负责什么？

A. 提供具体的业务功能实现
B. 运行LLM的应用程序环境，发起请求
C. 负责客户端与服务器之间的通信
D. 存储所有工具的定义和配置
正确答案：B

解析：MCP Host是运行LLM的应用程序环境（如Claude Desktop、Cursor IDE），负责发起对MCP Server的请求。

3. MCP支持的两种标准传输机制是什么？

A. HTTP GET和WebSocket
B. gRPC和REST API
C. stdio和SSE+HTTP POST
D. FTP和SMTP
正确答案：C

解析：MCP支持stdio（标准输入/输出）用于本地工具，以及SSE（Server-Sent Events）+ HTTP POST用于远程服务。

4. 与传统的Function Calling相比，MCP最主要的优势是什么？

A. 执行速度更快，延迟更低
B. 解耦模型与工具，实现标准化接口
C. 支持更多的编程语言和框架
D. 训练成本更低，部署更简单
正确答案：B

解析：MCP通过标准化协议实现模型与工具的解耦，使任意模型可与任意工具交互，而无需硬编码绑定，这是其最核心的优势。

5. MCP的安全机制包含以下哪些方面？

A. 用户同意与控制、数据隐私、工具安全、LLM采样控制
B. 自动加密传输、强制双因素认证、IP白名单限制

D. 硬件安全模块、生物特征识别、区块链存证
正确答案：A

解析：MCP采用多层安全机制，包括用户同意与控制、数据隐私保护、工具安全管理和LLM采样控制。

6. 在MCP架构中，Client组件的角色是什么？

A. 提供具体的业务功能实现
B. 内置于Host中，负责与Server通信

正确答案：B

解析：MCP Client内置于Host中，负责与MCP Server建立连接和通信，每个Client与一个Server建立1:1连接。
