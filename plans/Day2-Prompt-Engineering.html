Day 2：Prompt Engineering - 提示工程核心技巧

欢迎来到AI技能学习计划的第二天！今天我们将深入探索Prompt Engineering（提示工程）——这是与大模型高效交互的核心技能。作为后端工程师，掌握这些技巧将极大提升你构建AI应用的能力。

一、概念引入：什么是Prompt Engineering？

提示工程（Prompt Engineering）是通过精心设计和优化输入提示（Prompt），引导大模型产生高质量、稳定、符合预期的输出的技术。它本质上是一种"与AI高效沟通"的工程方法。

为什么Prompt Engineering如此重要？

  • 质量提升: 好的提示能显著提高输出准确性、相关性和一致性
  • 成本优化: 减少无效Token消耗，提升任务完成效率
  • 稳定性保障: 降低输出随机性，确保生产环境可靠性
  • 场景适配: 让通用大模型适配特定业务需求

1.1 提示工程的核心原则

有效的提示通常遵循以下原则：

  1. 清晰明确: 指令要具体，避免模糊和多义
  2. 结构化: 使用格式约束（如JSON、XML）规范输出
  3. 上下文充足: 提供必要的背景信息和示例
  4. 角色设定: 赋予模型特定身份，引导其专业领域输出

二、核心技巧：六大Prompt Engineering方法

2.1 角色设定（Role Playing）

通过明确的角色描述，让模型以特定身份思考：

  • 基础格式: "你是一个资深的Python后端架构师..."
  • 增强版本: "假设你是拥有10年经验的DevOps专家，擅长Kubernetes和微服务架构..."

2.2 思维链（Chain-of-Thought, CoT）

引导模型逐步思考，提升复杂问题解决能力：

  • 基础CoT: "让我们逐步思考这个问题..."
  • Few-shot CoT: 提供带思考过程的示例

2.3 结构化输出约束

通过格式约束确保输出可程序化处理：

  • JSON格式: "请以JSON格式返回，包含name、score、reason三个字段"
  • XML格式: "使用XML格式，根节点为response"
  • Markdown表格: "用Markdown表格呈现对比结果"

2.4 Few-shot示例

提供少量示例，让模型快速学习任务模式：

示例1:
输入: "苹果"
输出: {"category": "水果", "color": "红色", "taste": "甜"}

示例2:
输入: "胡萝卜"
输出: {"category": "蔬菜", "color": "橙色", "taste": "微甜"}

现在请处理:
输入: "香蕉"
输出:

2.5 参数调优技巧

结合API参数获得理想输出：

场景 参数设置 效果
事实性问答 temperature=0.1, top_p=0.9 输出稳定，减少随机性
创意写作 temperature=0.8, top_p=0.95 增加多样性，激发创意
代码生成 temperature=0.2, top_p=0.9 保持代码正确性和一致性

2.6 多轮对话上下文管理

有效管理对话历史，保持上下文一致性：

  • 上下文窗口: 注意Token限制，适时总结或截断
  • 系统消息: 使用system角色设定持久化指令
  • 历史总结: 长对话中插入总结性消息，释放Token空间

import json
import requests

class PromptEngineeringDemo:
    """提示工程演示类 - 展示多种高级提示技巧"""
    
    def __init__(self, api_key, base_url="https://dashscope.aliyun.com/api/v1"):
        self.api_key = api_key
        self.base_url = base_url
        self.headers = {
            "Authorization": f"Bearer {api_key}",
            "Content-Type": "application/json"
        }
    
    def call_qwen_api(self, messages, model="qwen-max", temperature=0.7, max_tokens=500):
        """调用Qwen API（通用封装）"""
        endpoint = f"{self.base_url}/chat/completions"
        
        payload = {
            "model": model,
            "messages": messages,
            "temperature": temperature,
            "max_tokens": max_tokens,
            "stream": False
        }
        
        try:
            response = requests.post(
                endpoint,
                headers=self.headers,
                json=payload,
                timeout=30
            )
            response.raise_for_status()
            return response.json()
        except requests.exceptions.RequestException as e:
            print(f"API调用失败: {e}")
            return None
    
    def demo_role_playing(self):
        """技巧1：角色设定"""
        print("=== 技巧1：角色设定示例 ===")
        
        messages = [
            {
                "role": "system",
                "content": "你是一位资深的技术面试官，专注于后端开发和系统设计。你的风格是专业、严谨但友好。"
            },
            {
                "role": "user",
                "content": "请为一个有3年经验的Java工程师设计一道系统设计面试题，并给出评估要点。"
            }
        ]
        
        response = self.call_qwen_api(messages, temperature=0.3)
        if response:
            content = self.extract_content(response)
            print(content)
        return response

{

需要分析的函数：
def process_data(data_list):
    result = []

        # 复杂处理逻辑
        processed = item * 2 + 5
        result.append(processed)
    return result"""
            }

        response = self.call_qwen_api(messages, temperature=0.1, max_tokens=600)

            content = self.extract_content(response)

    def demo_parameter_tuning(self):
        """技巧5：参数调优对比"""
        print("\n=== 技巧5：参数调优对比 ===")
        
        base_prompt = "请用Python实现一个快速排序算法"
        
        # 低temperature（稳定但缺乏创意）
        print("1. temperature=0.1（稳定输出）:")

        response_low = self.call_qwen_api(messages, temperature=0.1, max_tokens=400)
        if response_low:
            content = self.extract_content(response_low)
            print(f"输出长度: {len(content)}字符")

        print("\n2. temperature=0.9（创意输出）:")

        message = choice.get("message", {})
        return message.get("content", "")

四、实战练习：提示工程优化挑战

  1. 基础任务：将模糊提示优化为清晰提示
    • 原始提示："帮我处理数据"
    • 优化目标：明确数据格式、处理逻辑、输出要求
  2. 中级任务：设计一个客服场景的提示模板
    • 要求：包含角色设定、话术规范、异常处理指引
    • 输出：完整的对话系统提示
  3. 高级任务：构建一个代码审查的Few-shot提示
    • 要求：提供3个代码审查示例（包含问题类型、描述、建议）
    • 目标：让模型学会一致的代码审查模式

五、选择题测验（答案默认隐藏）

1. 关于Prompt Engineering的核心价值，以下哪项描述最准确？

A. 主要是为了减少API调用次数
B. 通过优化提示提升输出质量、稳定性和成本效率
C. 让大模型能够处理原本不能处理的任务类型
D. 主要用于美化输出格式，提高可读性
正确答案：B

解析：Prompt Engineering的核心价值在于：1）提升输出质量：通过清晰指令获得更准确、相关的回答；2）提高稳定性：降低随机性，确保生产环境可靠性；3）优化成本效率：减少无效Token消耗，提升任务完成效率。选项A是次要效果，选项C错误（Prompt Engineering不能扩展模型能力边界），选项D过于片面。

2. 在角色设定（Role Playing）技巧中，以下哪项是最佳实践？

A. 使用模糊的身份描述，如"你是一个专家"
B. 详细定义角色背景、专业领域和沟通风格
C. 只在系统消息中设定一次角色，后续对话不再提及
D. 为每个用户消息都重新设定不同角色
正确答案：B

解析：有效的角色设定应包含：1）具体身份（如"资深Python后端架构师"）；2）专业领域和资历（如"10年经验"）；3）沟通风格（如"专业、严谨但友好"）。选项A太模糊，选项C错误（长对话中可能需要强化角色），选项D会导致上下文混乱。

3. 思维链（Chain-of-Thought）提示的主要作用是什么？

A. 增加输出长度，让回答看起来更详细
B. 引导模型逐步推理，提升复杂问题解决能力
C. 减少API调用的Token消耗
D. 让模型跳过简单步骤，直接给出最终答案
正确答案：B

解析：思维链提示通过"让我们逐步思考..."等引导词，让模型展示推理过程，特别适用于需要多步推理的复杂问题（如数学题、系统设计、逻辑分析）。这能显著提升解决复杂问题的准确性。选项A是表面现象，选项C错误（CoT通常增加Token消耗），选项D与CoT目的相反。

4. 关于结构化输出约束，以下哪项描述是正确的？

A. 结构化输出会限制模型的创造力，应避免使用
B. 主要用于让输出看起来更美观整齐
C. 通过JSON/XML等格式约束，确保输出可程序化处理
D. 只适用于简单的数据提取任务
正确答案：C

解析：结构化输出（如JSON、XML、Markdown表格）的主要价值在于：1）程序化处理：便于后续代码解析和集成；2）一致性保障：确保关键字段完整；3）类型安全：明确数据类型和结构。选项A错误（结构化输出与创造力不冲突），选项B是次要效果，选项D错误（结构化输出适用于各种复杂任务）。

5. Few-shot学习提示中，示例选择的关键原则是什么？

A. 示例越多越好，尽可能覆盖所有可能性
B. 选择多样化、有代表性且高质量的示例
C. 优先选择简单易懂的示例，避免复杂案例
D. 随机选择示例，增加训练的随机性
正确答案：B

解析：Few-shot示例选择的关键是：1）代表性：覆盖任务的主要场景和边界情况；2）高质量：示例本身应正确、清晰、无歧义；3）一致性：示例间保持相同的格式和标准；4）适量：通常3-5个高质量示例优于大量低质量示例。选项A会导致Token浪费和示例冲突，选项C可能无法教会复杂任务，选项D会降低学习效果。

6. temperature参数在Prompt Engineering中的作用是什么？

A. 控制API调用的响应时间
B. 调整输出内容的专业程度
C. 控制输出的随机性和创造性
D. 限制模型的上下文记忆长度
正确答案：C

解析：temperature参数直接影响输出的随机性：1）低temperature（0.1-0.3）：输出稳定、可预测，适合事实性任务；2）中temperature（0.4-0.7）：平衡稳定性和创意；3）高temperature（0.8-1.0）：输出多样、有创意，但可能不稳定。选项A错误（不影响响应时间），选项B错误（专业程度由提示内容决定），选项D是上下文窗口限制。

7. 在多轮对话中，有效的上下文管理策略不包括：

A. 定期总结对话要点，减少历史Token占用
B. 将所有历史消息完整保留，确保不丢失任何信息
C. 适时清理无关或冗余的历史消息
D. 在长对话中插入系统消息强化角色设定
正确答案：B

解析：有效的上下文管理策略包括：1）适时总结：释放Token空间；2）清理冗余：移除不必要的历史消息；3）强化角色：在长对话中重新提示系统指令；4）关注相关：保留与当前任务最相关的历史。选项B错误：完整保留所有历史消息会快速耗尽上下文窗口，导致后续对话质量下降或无法继续。
